# אפליקציית מבחן אמצע

אפליקציה סטטית ל-GitHub Pages עבור מבחן אמריקאי בחשבונאות מתקדמת.

## מה האפליקציה עושה

- סטודנט מזין שם ותעודת זהות.
- מתקבלת בחינה של 11 שאלות:
  - 5 שאלות תזרים מזומנים.
  - 3 שאלות הלוואות והצמדות.
  - 3 שאלות אגרות חוב.
- לכל נושא נבחר אחד מתוך 3 סטים.
- סדר הנושאים, סדר השאלות וסדר התשובות מעורבבים.
- ההגרלה דטרמיניסטית לפי שם ותעודת זהות, כך שאותו סטודנט מקבל אותה גרסה גם אחרי רענון.
- ממשק המרצה מציג סטטיסטיקות, ציונים, סטים, התפלגות תשובות וייצוא CSV/JSON.

## הפעלה מקומית

פתח את הקובץ `index.html` בדפדפן.

## העלאה ל-GitHub Pages

1. צור מאגר GitHub חדש.
2. העלה אליו את כל תוכן התיקייה `exam-app`.
3. ב-GitHub עבור אל `Settings` ואז `Pages`.
4. בחר `Deploy from a branch`.
5. בחר ענף `main` ותיקיית `/root`.
6. לאחר הפרסום תקבל כתובת אתר.

## שמירת תשובות

GitHub Pages הוא אתר סטטי ולכן אינו שומר תשובות בשרת בעצמו.

כרגע האפליקציה שומרת הגשות ב-`localStorage` של הדפדפן שבו ההגשה בוצעה. זה טוב לבדיקה, אבל לא מספיק לאיסוף אמיתי מסטודנטים.

לאיסוף אמיתי מומלץ להשתמש ב-Google Sheet עם Apps Script:

1. צור Google Sheet חדש.
2. פתח `Extensions > Apps Script`.
3. הדבק את תוכן הקובץ `google-apps-script.gs`.
4. לחץ `Deploy > New deployment`.
5. בחר `Web app`.
6. הגדר:
   - Execute as: `Me`.
   - Who has access: `Anyone`.
7. העתק את כתובת ה-Web app.
8. בקובץ `data.js`, עדכן:

```js
googleScriptUrl: "PASTE_WEB_APP_URL_HERE"
```

9. העלה מחדש ל-GitHub.

## הערת אבטחה

מכיוון שמדובר באפליקציה סטטית, התשובות הנכונות נמצאות בצד הלקוח וסטודנט טכני מאוד יכול למצוא אותן. לתרגול זה מצוין. לבחינה רשמית עם אבטחה גבוהה צריך שרת אמיתי שמחזיק את מפתח התשובות.
